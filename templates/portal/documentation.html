Authentication

Secure your API requests with Bearer tokens

All API requests must include a valid Bearer token in the Authorization header.

Using cURL

curl -X POST "https://api.medisynth.io/v1/generate-patients" \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "population": 100,
    "state": "California"
  }'

Using JavaScript

const response = await fetch('https://api.medisynth.io/v1/generate-patients', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    population: 100,
    state: 'California'
  })
});

const data = await response.json();

Using Python

import requests

headers = {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
}

data = {
    'population': 100,
    'state': 'California'
}

response = requests.post(
    'https://api.medisynth.io/v1/generate-patients',
    headers=headers,
    json=data
)

Generate Patients

POST /v1/generate-patients

Generate realistic synthetic patient data with comprehensive medical histories.

Basic Parameters

Simple Request

{
  "population": 1000,
  "state": "California",
  "city": "San Francisco"
}

With Demographics

{
  "population": 500,
  "gender": "F",
  "ageMin": 25,
  "ageMax": 65,
  "state": "New York"
}

Parameter Reference

Core Parameters

  • population - Number of patients (1-10000)
  • seed - Random seed for reproducible results
  • outputFormat - "fhir", "ccda", "csv" (default: "fhir")

Demographics

  • gender - "M", "F", or null for mixed
  • ageMin - Minimum age (0-120)
  • ageMax - Maximum age (0-120)
  • state - US state name or abbreviation
  • city - City name within state

Response Format

{
  "status": "success",
  "jobId": "job_abc123def456",
  "message": "Generation job started successfully",
  "estimatedCompletionTime": "2024-01-15T10:30:00Z",
  "downloadUrl": "https://api.medisynth.io/v1/jobs/job_abc123def456/download",
  "webhookUrl": "https://your-app.com/webhook" // if provided
}

Advanced Options

Powerful features for complex scenarios

Custom Synthea Modules

Import your own Synthea modules to generate specific medical scenarios and conditions.

{
  "population": 500,
  "customModules": [
    "https://your-domain.com/modules/covid-variants.json",
    "https://your-domain.com/modules/rare-diseases.json"
  ],
  "state": "Texas",
  "ageMin": 18,
  "ageMax": 85
}

Target Specific Conditions

Generate patients with specific medical conditions using SNOMED-CT codes.

Single Condition

{
  "population": 200,
  "keepActiveConditions": [{
    "system": "SNOMED-CT",
    "code": "44054006",
    "display": "Type 2 diabetes mellitus"
  }]
}

Multiple Conditions

{
  "population": 300,
  "keepActiveConditions": [{
    "system": "SNOMED-CT",
    "code": "38341003",
    "display": "Hypertension"
  }, {
    "system": "SNOMED-CT", 
    "code": "13644009",
    "display": "Hypercholesterolemia"
  }],
  "keepLogic": "AND"
}

Output Formats

FHIR (Default)

Industry-standard healthcare data format

{
  "outputFormat": "fhir"
}

CCDA

Clinical Document Architecture

{
  "outputFormat": "ccda"
}

CSV

Tabular data for analysis

{
  "outputFormat": "csv"
}

Error Handling

Understanding API responses and troubleshooting

HTTP Status Codes

200 Success - Job created successfully
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API token
429 Rate Limited - Too many requests

Error Response Format

{
  "error": {
    "code": "invalid_parameters",
    "message": "Invalid age range specified",
    "details": {
      "field": "ageMax",
      "reason": "Maximum age cannot exceed 120",
      "provided": 150,
      "allowed": "0-120"
    },
    "requestId": "req_abc123def456"
  }
}
